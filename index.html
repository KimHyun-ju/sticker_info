Introduction

This program is a sticker that allows you to place multiple stickers on the desktop in the desired location, in the desired size, and in the desired direction.


Context Menu Description

Menu

Image

Description

Change

Image

Change the image of the widget with the context menu open (also possible by dragging and dropping : Tutorial 1).

A dialog window opens where you can select an image file.

 

*caution

-Avoid using images that are too large (larger than the screen resolution).

Change

Image

by

Area

Change the image of the widget with an area screenshot.

 

*caution

This feature is unstable.

Do not use for monochromatic or subjectless images.

If the image is broken or not visible, delete the sticker.db file from the path where the executable file is located.

Add

widget

This is tutorial number 3.

 

Create a new basic sticker in the center of the screen.

Resize

&

Rotate

widget

Sticker resizing and rotation functions.

The rotation function is somewhat unreasonable.

Function

Setting

On Click

This function defines the action performed when clicked.

 

1. Enter the webpage address

ex input : www.google.com

2. You can create a shortcut by copying the path (shift + right click) ex input : "C:\"

3. Ability to write Python3 code. ex : print("hello world")

4. Run the Python code file (.py).


Click lock will be disabled when you turn on the function.

Unlock

Click

(Lock

Click)

You can disable or lock the click.

Delete

This

Widget

Delete a sticker with the context menu open. Deleted stickers are removed from the database.

Delete

All

Widget

Delete all stickers. This is the same as resetting the sticker.

Help

Open the help page.

Save As

This

Sticker


Save the sticker with the context menu open as a picture file.

Quit

This

Widget


Turns off the current sticker. Since it is not deleted from the db, it will be created again the next time you turn on the program.

Quit

All

Widget

Turn off all stickers and close the program. Since they are not deleted from the db, when the program is run again, all stickers are created in their previous state.
